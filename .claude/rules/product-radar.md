## What to Capture

### Always document when:
- A technical module is built, configured, or taught (Claude.ai, Claude Code, Anti-gravity)
- A worker or agent is designed, structured, or deployed
- A site, landing page, or budget is created
- A new tool, MCP, or integration is configured
- A CLI command sequence solves a non-obvious problem
- An architecture decision is made (and WHY)
- A workflow pattern emerges (n8n, ManyChat, Instagram DMs, etc.)
- A prompt structure or agent persona is defined
- A problem is solved that took significant iteration to crack
- A concept, framework, or methodology is articulated
- Any conversation generating strategic or educational insight

### Capture format (append to Obsidian):
```
## [YYYY-MM-DD] — [Session Theme]

**What was built/decided:**
[1-3 sentences]

**Product module this feeds:**
[Core Modules / Squad Architecture / Automation / Content / Business Systems / AI Workers / Standalone Offer → which product?]

**Raw material value:**
[Why this is worth teaching — what problem it solves, how long it would take someone to figure out alone]

**Key artifacts:**
- [file paths, commands, prompts, concepts, decisions]
```

## Agent Behavior Rules

### During every interaction:
1. **Identify** — Is this interaction generating product-worthy content?
2. **Flag** — If yes, mention to user: _"Isso é material para RUNA SYSTEMS — vou documentar."_
3. **Document** — Append capture entry to vault file before ending the session

### Product opportunity triggers (any of these = capture):
- User says "vou construir", "preciso criar", "como faço", "bora implementar"
- A new file, workflow, config, or agent is created
- A concept is explained in depth
- A problem is debugged with a non-obvious solution
- A business or content strategy decision is made
- A standalone product or offer is designed
- A client journey or upsell path is mapped

### Never skip capture for:
- "Simple" tasks — a 5-minute setup can take someone else 5 hours
- Repeatable commands — every CLI sequence is a potential step-by-step
- Failures and pivots — what didn't work is as valuable as what did
