# Story 5.1 — Create MAYA Agent (Audio-Visual Production)

## Story Statement

As ORION (orchestrator), I want a formal MAYA agent definition activatable via `@maya`,
so that all audio-visual production tasks have a dedicated, scoped agent with clear
authority boundaries and inference.sh skill assignments.

## Background

The Runa Systems ecosystem lacked a dedicated agent for media production. FREYJA was
handling both narrative intelligence and AV generation — two fundamentally different
concerns. MAYA is the new AV Production specialist who receives briefs from FREYJA
and executes all inference.sh media generation tasks.

MAYA = Sanskrit/Maya cosmology: "the creative force that weaves material reality" —
perfectly aligned with audio-visual production (creating visual/audio reality from briefs).

## Acceptance Criteria

- [ ] **AC1:** `@maya` activates the agent with full persona (icon 🎬, correct greeting)
- [ ] **AC2:** Agent definition includes all inference.sh AV skills in `whenToUse`

- [ ] **AC4:** `maya/MEMORY.md` created at `.aiox-core/development/agents/maya/MEMORY.md`

### FREYJA→MAYA Handoff Protocol
```
FREYJA outputs brief with:
  - content_type: image|video|voice|music
  - narrative_direction: string (what this asset must convey)
  - style_reference: ARCHITECT|MANIFESTO|TERMINAL (or custom)
  - technical_spec: dimensions, duration, format
  - approval_required: true (always)

MAYA outputs:
  - asset_url: string
  - asset_type: string
  - generation_params: object (for reproducibility)
  - ready_for_review: true
```
