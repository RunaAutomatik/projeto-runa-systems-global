# Story 1.4 — Deploy content-worker to Railway

## Story Statement

As ORION (operator), I want the content-worker running on Railway with a public URL,
so that FREYJA can trigger content rendering from anywhere without
the local machine being online.

## Background

The content-worker Express server runs on port 3001 locally. A `railway.json` config
already exists at `apps/content-worker/railway.json` with NIXPACKS builder and
`/health` healthcheck configured. The service just needs to be linked and deployed.

This is a @devops exclusive task (git push / Railway management authority per agent-authority.md).

## Acceptance Criteria

- [ ] **AC1:** content-worker is deployed on Railway and accessible via public HTTPS URL
- [ ] **AC2:** `GET {public-url}/health` returns `{ status: 'ok', timestamp: '...' }`
- [ ] **AC3:** All required environment variables are configured in Railway dashboard
- [ ] **AC4:** `POST {public-url}/render` with a valid carousel brief returns image URLs
- [ ] **AC5:** Service restarts automatically on failure (already configured in railway.json)
- [ ] **AC6:** Railway deploy is triggered from `apps/content-worker/` subdirectory (monorepo root source)

## Dev Notes

### Existing Railway Config

`apps/content-worker/railway.json` — already correct, no changes expected:

```json
{
  "build": { "builder": "NIXPACKS" },
  "deploy": {
    "startCommand": "node src/index.js",
    "healthcheckPath": "/health",
    "healthcheckTimeout": 30,
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 3
  }
}
```

### Required Environment Variables

| Variable                   | Where it comes from            |
|----------------------------|--------------------------------|
| `SUPABASE_URL`             | Supabase project settings      |
| `SUPABASE_SERVICE_ROLE_KEY`| Supabase project settings      |
| `NODE_ENV`                 | Set to `production`            |
| `PORT`                     | Injected by Railway (automatic)|

### Monorepo Railway Pattern

Railway supports monorepos via "Root Directory" setting per service.
Set root to `apps/content-worker` — Railway will run NIXPACKS from there,
finding `package.json` and `railway.json` in that subdirectory.

### FFmpeg on Railway (relevant after Story 1.1)

When Story 1.1 is merged, FFmpeg must be available in the Railway environment.
Add `nixpacks.toml` at `apps/content-worker/nixpacks.toml`:

```toml
[phases.setup]
nixPkgs = ["ffmpeg"]
```

This is not blocking for Story 1.4 if Story 1.1 is not yet merged.

### .env Note

Local `.env` is at repo root and loaded via `dotenv` with relative path resolution.
In Railway, env vars are injected directly — `dotenv` call will not find the file
but will NOT crash (dotenv silently continues if file missing). Verify this works
by checking Railway logs for any "missing env" errors after deploy.
